Skater Model

Andrew Bloese - April 29, 2026

Description:

Overview

In this program, I implemented a hierarchical model of a human figure riding a skateboard in a circular path. The per-face textures for each node of the tree are all generated upon initialization of a model. In addition to generated textures for the clothing and hair / skin, the model supports per-face tinting, so colors of clothing articles or hair or skin tone can be easily applied using tints. Each node is an "instance" of a cube prototype, each with a scale (x,y,z) , translation (x,y,z) , rotation (x,y,z). An additional pivot (x,y,z) attribute was used to allow for adjusting where the rotation point lies relative to the node's center.

Textures

The textures used in the model were generated using various helper functions in Bloese_Textures.js. The hair function generates random grayscale strips, so that a tint color can be applied in the shader. The flannel shirt function generates a flannel pattern in greyscale, so a tint can be applied to determine the shirt color. Similarly, the other textures use a similar strategy by generating the raw RGBA bytes for the texture once on model initialization. Per-face textures and tints were achieved by providing each node with an array of textures in the same order as the faces of the prototype cube.

Phong Shading

The avatar textures and floor texture are treated as decals, so a tint can be applied to a texture lookup. For the red, green, blue and magenta obstacles, phong shading was utilized by defining a material for each color. Flags are sent to the GPU via uniforms to determine if a texture or material or debug texture are being applied. The debug texture is the "normal" color mixed with the texture coordinates of each face, and it resembles an oil slick type texture. One obstacle was left untextured and without material to show this.

Camera Modes

By using the camera controls the user can toggle between various camera modes.

  • Fixed Camera - is like having a camera setup on a tripod watching the avatar skate around
  • Follow Camera (Front and Back) - the camera follows the avatar either from the front or the back, looking at the avatar
  • Orbit Camera - this camera slowly orbits in the opposite direction of the avatar while focusing on the world origin (0,0,0)

Fun Quirks

  • Container Nodes - By using a node with scale of 0, and ignoring render on nodes with 0 scale, it is possible to create "groups" of nodes
    • In order to easily place the beard pieces, a facial hair container node was used
    • To proprerly group the chest (original root) and skateboard nodes, an "avatar" container node was created to act as a root.
  • Grayscale Textures - By using grayscale textures, it is possible to use the builtin mix() function in the fragment shader to change the color using a tint color uniform.
    • This enables easy customization of a model's skin tone, hair color, or clothing.
